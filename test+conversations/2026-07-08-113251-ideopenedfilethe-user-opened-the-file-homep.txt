  The recipe

  I made plot_reflection.py take optional arguments so it works for either
  model:

  python plot_reflection.py <data.dat> <out.png> <model-label>

  So to plot Debye:

  gcc -Wall -O2 -o debye Debye.c -lm && ./debye        # writes reflection.dat

  Two things to be aware of, which is why I passed explicit arguments rather
  than relying on defaults:

  1. Debye.c and Drude.c both write the same reflection.dat. Whichever binary
  you run last owns that file. Each .dat already contains the right analytic 
  column for its model (Debye's is ОөвҲһ + Оөd/(1+jПүП„), Drude's is ОөвҲһ вҲ’ 
  ПүpВІ/(ПүВІвҲ’jgПү)), so just plot right after you run вҖ” don't run one binary and
  plot expecting the other. Writing to reflection_debye.png /
  reflection_drude.png keeps the two figures side by side.
  2. Python env: your base and ai_physics conda envs don't have
  numpy/matplotlib; the autograd env does. Plain python plot_reflection.py fails
  with ModuleNotFoundError. Use:
  ~/miniconda3/envs/autograd/bin/python plot_reflection.py reflection.dat
  reflection_debye.png Debye
  2. (or conda activate autograd first).

  With no arguments it still defaults to reflection.dat вҶ’ reflection.png (title
